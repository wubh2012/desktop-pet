# Live2D Cat Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a Live2D Tororo white-cat renderer while preserving the current Three.js renderer, and expose a local HTTP command API.

**Architecture:** Keep Electron main as the owner of windows, tray, IPC, and the local HTTP server. Add shared command validation so tray, HTTP, and renderer use the same command vocabulary. Add a Live2D renderer module that can load `public/live2d/tororo/tororo.model3.json` and react to the existing action IPC.

**Tech Stack:** Electron, Vite, TypeScript, Vitest, PixiJS v6, pixi-live2d-display, Node `http`.
